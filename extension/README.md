# Auth Proxy Switcher — расширение для Chrome

Минимальное расширение (Manifest V3, без CSS), которое настраивает HTTP/HTTPS-прокси
и **автоматически подставляет `login:password`** в ответ на запрос авторизации прокси
(HTTP 407) — браузер не показывает системное окно ввода.

Работает в паре с прокси из этого репозитория (`proxy.js`).

## Установка

1. Открой `chrome://extensions`.
2. Включи **Developer mode** (справа сверху).
3. Нажми **Load unpacked** и выбери папку `extension/`.

## Сборка в .zip

Чтобы получить архив для загрузки в Chrome Web Store или для распространения:

```bash
npm run build:extension
# или: bash scripts/build-extension.sh
```

Скрипт возьмёт версию из `manifest.json` и создаст
`dist/auth-proxy-switcher-<version>.zip` в корне репозитория. В архив попадают
только рабочие файлы (`manifest.json`, `background.js`, `popup.html`, `popup.js`)
без вложенной папки — как того ожидает Chrome.

## Использование

1. Кликни по иконке расширения.
2. Заполни поля:
   - **Scheme** — `http` или `https` (транспорт до самого прокси);
   - **Host** / **Port** — адрес прокси (например `proxy.example.com` : `8080`);
   - **Login** / **Password** — учётные данные.
3. Отметь **Enable proxy** и нажми **Save**.

`localhost`, `127.0.0.1` и `::1` идут напрямую (в bypass-листе), чтобы браузер
оставался рабочим. Чтобы выключить прокси — сними галочку **Enable proxy** и сохрани.

## Как это устроено

- `background.js` (service worker):
  - `chrome.proxy.settings` — применяет/сбрасывает конфигурацию `fixed_servers`;
  - `chrome.webRequest.onAuthRequired` — при `isProxy === true` возвращает
    `authCredentials` из сохранённых настроек.
- `popup.html` / `popup.js` — форма настроек, хранит данные в `chrome.storage.local`.

## Требуемые разрешения

| Разрешение              | Зачем                                            |
| ----------------------- | ------------------------------------------------ |
| `proxy`                 | задавать настройки прокси браузера               |
| `webRequest`            | перехватывать запрос авторизации                 |
| `webRequestAuthProvider`| отвечать на 407 своими учётными данными          |
| `storage`               | хранить настройки                                |
| `<all_urls>`            | прокси и авторизация применяются ко всем запросам |

> Пароль хранится в `chrome.storage.local` в открытом виде — это демонстрационное
> расширение. Для реального использования продумай защиту учётных данных.
